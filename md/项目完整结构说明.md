## 五、当前业务重点模块

### 1. AI 助手模块

- 页面：`pages/ai-assistant/index.vue`
- 配置：`config/ai.js`
- 服务：`services/ai.js`
- 静态资源：`static/ai/`

当前已支持：

- 灵鹿头像与背景图
- 聊天软件式双边对话 UI
- 用户头像与 AI 头像
- 百炼模型问答
- 会话缓存

### 2. 攻略/笔记模块

- 列表页：`pages/guides/index.vue`
- 发布页：`pages/guide-publish/index.vue`
- 详情页：`pages/guide-detail/index.vue`
- 服务层：`services/guides.js`
- 本地兼容：`common/published-guides.js`
- 后端接口：`server/index.js`

当前方向：从 mock/本地兜底，逐步转向数据库/API 驱动。

### 3. 徒步地图模块

- 页面：`pages/hiking/index.vue`
- 子组件：`pages/hiking/components/`
- Store：`stores/useHikingStore.js`
- 服务层：`services/hiking-native.js`
- 公共逻辑：`common/hiking-*`

这是当前项目中技术结构最复杂的模块之一，涉及原生桥接、H5 地图与兼容处理。

## 六、阅读项目建议顺序

如果是新接手开发，建议按下面顺序看：

1. `pages.json`：先看页面路由入口
2. `main.js`、`App.vue`：看全局挂载方式
3. `pages/home/index.vue`：看首页入口
4. `pages/guides/`、`pages/guide-publish/`、`pages/guide-detail/`：看攻略链路
5. `pages/ai-assistant/index.vue` + `services/ai.js`：看 AI 助手链路
6. `pages/hiking/` + `stores/useHikingStore.js`：看徒步地图复杂模块
7. `server/index.js` + `server/db.js`：看后端 API 与数据库
8. `docs/*.sql`：看完整数据结构

## 七、补充说明

- `node_modules/`、`unpackage/` 属于依赖和构建产物，体量较大，通常不作为业务梳理重点。
- `md/` 目录现在用于沉淀新的项目说明文档，便于后续持续补充。
- 根目录下多份中文 `.md` 文件主要是历史方案、排障记录和修复纪要，对理解项目演进很有帮助。

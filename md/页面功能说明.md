# 遇见新疆_uniapp 页面功能说明

## 一、文档说明

本文档用于说明 `F:\AI编程\遇见新疆_uniapp` 项目中各页面的主要功能、核心交互和对应业务定位，便于后续开发、联调和维护。

页面路由配置主要来源于：`pages.json`

当前项目核心页面包括：

- 首页
- 目的地列表页
- 目的地详情页
- 攻略/发现流页
- 攻略详情页
- 攻略发布页
- AI 助手页
- 徒步页
- 安全地图页
- 登录注册页
- 我的页
- 设置页
- 网页容器页

## 二、页面总览

| 页面 | 路径 | 作用 |
|---|---|---|
| 首页 | `pages/home/index` | 应用主入口，展示推荐景区、小众推荐和运营内容 |
| 目的地页 | `pages/destinations/index` | 景区列表、搜索、筛选入口 |
| 目的地详情页 | `pages/destination-detail/index` | 景区详情、地图、路线、安全、AI 提问入口 |
| 攻略页 | `pages/guides/index` | 笔记/攻略发现流，支持搜索、分类、排序、发布 |
| 攻略详情页 | `pages/guide-detail/index` | 查看图文/视频/纯文字笔记详情及互动信息 |
| 攻略发布页 | `pages/guide-publish/index` | 发布笔记，支持文字、图片、视频、定位、轨迹 |
| AI 助手页 | `pages/ai-assistant/index` | 与灵鹿对话，获取新疆旅行建议 |
| 徒步页 | `pages/hiking/index` | 徒步模式、轨迹记录、GPS 信息和安全守护 |

| 我的页 | `pages/account/index` | 个人资料、我的攻略、我的收藏 |
| 设置页 | `pages/settings/index` | 账号信息与系统设置入口 |
| 网页容器页 | `pages/live-webview/index` | 打开外部网页/直播搜索页 |

## 三、各页面功能说明

### 1. 首页 `pages/home/index.vue`

#### 页面定位

首页是整个 App 的第一入口，承担：

- 品牌展示
- 推荐内容分发
- 目的地导流
- AI 问答入口延伸

#### 主要功能

- 展示顶部品牌头图 `丝路疆寻`
- 显示精选目的地内容
- 显示小众推荐内容
- 支持点击景区卡片进入详情页
- 支持从卡片上直接发起 `Ask AI`
- 支持侧边菜单/页面交互

#### 页面价值

- 承担产品第一印象
- 作为景区内容分发页
- 连接 AI 助手和目的地详情链路

### 2. 目的地列表页 `pages/destinations/index.vue`

#### 页面定位

用于浏览新疆景区列表，是“找景区”的主入口。

#### 主要功能

- 顶部搜索景区、地区、分类
- 打开筛选面板
- 按分类、地区进行过滤
- 双列瀑布流展示景区卡片
- 点击卡片进入景区详情页
- 卡片上可直接发起 `Ask AI`

#### 典型使用场景

- 用户不知道去哪，先浏览景区
- 用户按地区/分类快速找目的地
- 用户从列表直接进入详情或 AI 咨询

### 3. 目的地详情页 `pages/destination-detail/index.vue`

#### 页面定位

这是景区内容最完整的页面，负责呈现单个景区的深度信息。

#### 主要功能

- 展示景区封面图、标题、地点、简介
- 展示景区介绍、历史、亮点
- 展示推荐季节、停留时长、适合人群
- 展示门票参考和开放时间
- 展示地图定位、路线模式、距离、时间、打车估算
- 支持打开地图导航
- 支持刷新当前位置和天气/路线信息
- 展示安全保障相关内容
- 可以与安全地图页联动
- 可以与 AI 助手页联动，围绕该景区提问

#### 页面价值

- 是景区数据展示的核心页
- 也是导航、安全、问答的中枢页

### 4. 攻略/发现流页 `pages/guides/index.vue`

#### 页面定位

这是新疆攻略和笔记的内容流页面，偏“小红书式发现流”。

#### 主要功能

- 顶部主 tab：`关注 / 发现 / 同城`
- 子分类 tab：`推荐 / 自驾 / 美食 / 安全 / 徒步 / 住宿`
- 搜索作者、标题、摘要、标签
- 打开排序面板选择排序方式
- 双列瀑布流展示攻略卡片
- 支持图文卡、视频卡、纯文字卡
- 显示作者头像、昵称、点赞、评论/收藏信息
- 点击卡片进入攻略详情页
- 点击左上聊天按钮进入 AI 助手页
- 点击右下 `发布` 进入发布页

#### 当前内容来源

- 优先从后端 API 读取
- 兼容本地发布数据/旧数据兜底

#### 页面价值

- 是 UGC/攻略内容消费主入口
- 承担发布回流和内容分发功能

### 5. 攻略详情页 `pages/guide-detail/index.vue`

#### 页面定位

查看单条攻略/笔记详情，支持图文、视频、纯文字三种内容形态。

#### 主要功能

- 顶部显示作者头像、昵称、关注按钮、分享按钮
- 若为视频内容，直接显示视频播放器
- 若为图文内容，显示图片轮播和页码指示器
- 若为纯文字内容，显示文字 hero 区块
- 展示标题、标签、发布时间
- 展示攻略正文/摘要
- 若笔记附带徒步轨迹，可展示轨迹地图、距离、时长、爬升
- 显示互动信息，如粉丝、关注、点赞等

#### 页面价值

- 是攻略消费链路的详情承接页
- 也是后续点赞、评论、收藏、关注扩展的基础页

### 6. 攻略发布页 `pages/guide-publish/index.vue`

#### 页面定位

这是用户发布新疆笔记/攻略的页面，当前已偏轻量笔记发布模式。

#### 主要功能

- 输入标题
- 输入简短描述
- 输入标签
- 添加图片或视频
- 自动判断内容类型：`图文 / 视频 / 文字`
- 自动推断子分类：`推荐 / 自驾 / 美食 / 安全 / 徒步 / 住宿`
- 自动获取定位
- 手动重新定位或选择附近位置
- 可附带徒步轨迹一起发布
- 顶部发布按钮和提交逻辑
- 发布成功后写入后端数据库并回流到攻略流

#### 当前发布内容支持

- 纯文字笔记
- 图文笔记
- 视频笔记
- 带徒步轨迹的笔记

#### 页面价值

- 是 UGC 内容生产主入口
- 连接用户账号、媒体上传、数据库存储、发现流回流

### 7. AI 助手页 `pages/ai-assistant/index.vue`

#### 页面定位

这是“灵鹿”AI 旅游助手页面，负责旅行咨询和上下文问答。

#### 主要功能

- 显示灵鹿头像和背景图
- 显示当前连接状态
- 若从景区/其他页面带入上下文，可显示当前接入上下文卡片
- 提供快捷问题列表
- 用户与 AI 进行聊天式对话
- 支持用户头像和 AI 头像双边消息布局
- 支持 AI 流式输出动画效果
- 支持清空会话
- 支持将外部页面 prompt 自动填入对话框或直接发问

#### 当前能力来源

- 使用阿里百炼兼容接口
- Key 已在 `config/ai.js` 中配置

#### 页面价值

- 作为全局智能导览入口
- 可从首页、目的地页、详情页等多个页面跳转进入

### 8. 徒步页 `pages/hiking/index.vue`

#### 页面定位

徒步页是项目里最复杂的工具页，面向真实户外场景。

#### 主要功能

- 显示 GPS/定位状态
- 显示经纬度、海拔、精度、累计距离
- 显示在线/离线状态
- 显示日落倒计时和风险级别
- 显示安全守护状态
- 显示地图舞台
- 支持刷新定位、回到当前位置
- 支持开始/停止记录轨迹
- 支持完成轨迹、清空轨迹
- 支持开启/关闭守护模式
- 支持 SOS 短信联动

#### 子组件拆分

- `HikingHeaderPanel.vue`
- `HikingMapStage.vue`
- `HikingBottomControls.vue`

#### 页面价值

- 承担徒步场景的功能中台
- 可与攻略发布页联动，发布轨迹型笔记

### 9. 安全地图页 `pages/safety-map/index.vue`

#### 页面定位

为景区提供安全参考图和应急信息。

#### 主要功能

- 展示景区安全图
- 支持在线/离线两种安全图模式
- 显示地图风险等级
- 展示地形风险、天气风险、信号风险
- 展示推荐安全路线
- 展示服务点、集合点信息
- 展示应急联系方式和提示说明

#### 页面价值

- 面向景区安全和应急场景
- 与目的地详情页形成补充关系

### 10. 登录注册页 `pages/auth/index.vue`

#### 页面定位

提供账号登录和注册能力。

#### 主要功能

- 登录/注册模式切换
- 注册时输入昵称、邮箱、密码、确认密码
- 登录时输入邮箱和密码
- 表单校验
- 调用后端登录/注册接口
- 保存登录态到本地
- 成功后跳转回业务页面

#### 页面价值

- 是账号体系的入口页
- 支撑发布、收藏、个人资料等所有用户态功能

### 11. 我的页 `pages/account/index.vue`

#### 页面定位

展示用户个人信息和个人内容资产。

#### 主要功能

- 显示用户头像、昵称、邮箱
- 支持点击头像修改头像
- 支持修改昵称
- 显示个人统计信息
- 已登录时展示“我的攻略”列表
- 已登录时展示“我的收藏”列表
- 未登录时展示登录/注册引导卡片
- 进入设置页

#### 页面价值

- 是用户中心页
- 承接个人资料与个人内容管理

### 12. 设置页 `pages/settings/index.vue`

#### 页面定位

用于账号信息维护和系统设置入口聚合。

#### 主要功能

- 自定义导航栏
- 已登录时显示账号信息区
- 支持修改昵称
- 展示账户设置项：旅行足迹、消息通知、语言设置、通用设置
- 支持退出登录

#### 页面价值

- 补充“我的页”的设置能力
- 承担账户管理入口

### 13. 网页容器页 `pages/live-webview/index.vue`

#### 页面定位

用于打开外部网页内容，比如直播搜索页或第三方页面。

#### 主要功能

- 接收 url 参数
- 接收页面标题参数
- 显示顶部返回栏
- 使用 `web-view` 打开指定链接

#### 页面价值

- 作为外链承载页
- 避免直接跳出 App 上下文

## 四、页面之间的主要链路

### 1. 首页链路

- 首页 -> 目的地详情页
- 首页 -> AI 助手页
- 首页 -> 目的地列表页

### 2. 景区链路

- 目的地列表页 -> 目的地详情页
- 目的地详情页 -> 安全地图页
- 目的地详情页 -> AI 助手页

### 3. 攻略链路

- 攻略流页 -> 攻略详情页
- 攻略流页 -> 发布页
- 发布页 -> 数据库存储 -> 回流攻略流/详情页

### 4. 用户链路

- 未登录 -> 登录注册页
- 登录后 -> 我的页 / 设置页 / 发布页

### 5. 徒步链路

- 徒步页记录轨迹
- 发布页可附带徒步轨迹发布
- 攻略详情页可展示附带轨迹

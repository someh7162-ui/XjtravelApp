# 遇见新疆_uniapp 颜色与组件规范说明

## 一、文档目的

这份文档用于整理当前项目已经形成的颜色系统、通用组件视觉规则和页面复用规范，方便后续继续开发时保持风格统一，避免每个页面各写一套样式。

适用范围：

- 首页与目的地类页面
- AI 助手页
- 我的页、设置页、登录页等账户类页面
- 后续新增的攻略、笔记、徒步扩展页面

## 二、当前颜色系统

项目主色变量定义在：`uni.scss`

```scss
$theme-color: #c44536;
$theme-secondary: #e8a87c;
$theme-accent: #d4a574;
$theme-bg: #faf8f5;
$theme-text: #2d1810;
$theme-muted: #8b6f47;
$theme-border: rgba(196, 69, 54, 0.15);
```

### 1. 颜色角色说明

| 变量 | 颜色值 | 角色 | 建议用途 |
| --- | --- | --- | --- |
| `$theme-color` | `#c44536` | 主品牌色 | 主按钮、高亮文字、激活态、主标签 |
| `$theme-secondary` | `#e8a87c` | 辅助暖色 | 渐变中段、强调背景、提示块 |
| `$theme-accent` | `#d4a574` | 金棕强调色 | 轻强调、装饰色、图片占位底色 |
| `$theme-bg` | `#faf8f5` | 页面底色 | 大页面背景、浅色区块背景 |
| `$theme-text` | `#2d1810` | 主文字色 | 标题、正文、重要信息 |
| `$theme-muted` | `#8b6f47` | 次级文字色 | 描述文案、辅助信息、未激活态 |
| `$theme-border` | `rgba(196, 69, 54, 0.15)` | 弱边框色 | 卡片描边、输入区描边、分隔 |

### 2. 整体色彩方向

项目目前不是互联网常见的冷白蓝紫风，而是明显偏新疆旅行场景的暖色系统：

- 主视觉偏砖红、暖橙、沙金
- 背景偏米白，不使用纯冷白
- 文字偏深棕，不使用纯黑
- 阴影偏轻，避免厚重的商务后台感

这套方向适合：

- 旅游内容
- 风景图片展示
- 人文氛围页面
- AI 助手的陪伴型角色设定

## 三、颜色使用规范

### 1. 主色使用规范

`$theme-color` 应优先用于：

- 主要 CTA 按钮
- 当前选中态
- 重要标签
- 强提醒但非报错信息

不建议：

- 大面积整屏纯主色铺满
- 长段正文文字都使用主色
- 多个主按钮连续堆叠，造成视觉噪声

### 2. 背景使用规范

- 页面底层优先使用 `$theme-bg`
- 卡片优先使用白色背景 + 弱边框 + 轻阴影
- 顶部 hero 区允许使用暖色渐变、图片叠蒙层、局部光感效果

推荐组合：

```scss
background: $theme-bg;
background: #ffffff;
background: linear-gradient(135deg, $theme-color 0%, $theme-secondary 55%, $theme-accent 100%);
```

### 3. 文字颜色规范

- 标题、核心数据、重点内容：`$theme-text`
- 次级描述、辅助说明、未激活文案：`$theme-muted`
- 深色背景上的文字：优先白色

避免：

- 页面里同时出现过多不同深浅的灰色
- 浅背景上使用过浅文字，导致可读性不足

### 4. 边框与阴影规范

项目整体更偏内容展示，不适合重描边和厚阴影。

推荐延续当前做法：

```scss
border: 2rpx solid $theme-border;
box-shadow: 0 16rpx 40rpx rgba(45, 24, 16, 0.08);
```

规则：

- 描边用于增强卡片边界，不抢主视觉
- 阴影用于抬升层级，不做悬浮夸张效果
- 同一页面阴影强度尽量控制在 1 到 2 个等级内

## 四、全局通用样式基座

全局基础样式文件：`common/common.scss`

当前已经沉淀出的通用 class：

- `.page-shell`：页面根容器
- `.page-scroll`：主滚动容器
- `.hero-gradient`：暖色渐变头图基底
- `.section`：统一左右边距
- `.card`：统一卡片容器
- `.section-title`：统一分区标题
- `.muted-text`：次级文字
- `.pill`：胶囊形基础结构
- `.tag-primary`：主色标签
- `.bottom-space`：底部留白
- `.cover-image`：通用图片铺满

这些 class 应视为样式规范的第一层，后续页面优先复用，不建议重复发明近似 class。

## 五、组件视觉规范

## 1. 卡片 `card`

基础来源：`common/common.scss`

```scss
.card {
  background: #ffffff;
  border: 2rpx solid $theme-border;
  border-radius: 28rpx;
  box-shadow: 0 16rpx 40rpx rgba(45, 24, 16, 0.08);
}
```

适用场景：

- 内容信息块
- 设置菜单块
- 空状态块
- 统计卡片
- 弹层式内容容器

规范：

- 默认圆角以 24rpx 到 32rpx 为主
- 不同页面可以改内边距，但不建议把卡片改成完全不同的边框语言
- 如果是图片卡片，可在内部再叠加遮罩，不要直接破坏卡片基底规范

## 2. 胶囊类组件 `pill`

基础来源：`common/common.scss`

```scss
.pill {
  display: inline-flex;
  align-items: center;
  justify-content: center;
  padding: 12rpx 24rpx;
  border-radius: 999rpx;
}
```

适用场景：

- 筛选标签
- 快捷问题
- 状态标签
- 小型功能入口

规范：

- 轻交互类胶囊优先用浅底色 + 深文字
- 强调胶囊可以叠加 `.tag-primary`
- 文字过长时不要强撑单行，应限制内容长度或做两行策略

## 3. 分区标题 `section-title`

基础来源：`common/common.scss`

```scss
.section-title {
  font-size: 38rpx;
  font-weight: 600;
  line-height: 1.4;
}
```

规范：

- 页面一级分区标题优先复用这个 class
- 如果页面需要更强的视觉层次，可额外加自定义 class，例如首页的 `editorial-title`
- 不建议每个页面单独再定义一套完全不同的标题基准字号

## 4. 底部导航组件 `components/AppTabBar.vue`

当前特点：

- 固定定位悬浮在底部
- 外层使用安全区补偿
- 内层复用 `.card` 样式形成胶囊感
- 激活态只切换文字颜色，不做过重的按钮块

适配规范：

- 新增 tab 项时保持字数精简
- 单个 label 最好控制在 2 到 4 个字
- 不建议在 tabbar 中加入过多辅助文案或徽标，避免挤压布局

## 5. 图片组件 `components/CachedImage.vue`

当前视觉策略：

- 默认浅暖色占位背景
- 加 shimmer 骨架动画
- 图片统一铺满容器

适合场景：

- 景区卡片
- 攻略封面
- 头像和背景图之外的大多数内容图片

规范：

- 列表图片优先复用 `CachedImage`
- 图片容器负责圆角和裁切，图片本身尽量只负责显示
- 加载失败时应接受降级为空状态，不要因为单图失败阻塞整体布局

## 六、页面模块规范

### 1. Hero 区

项目多个页面已经采用顶部 hero 结构，如：

- `pages/home/index.vue`
- `pages/destinations/index.vue`
- `pages/account/index.vue`
- `pages/auth/index.vue`
- `pages/ai-assistant/index.vue`

通用建议：

- 顶部区承担情绪与品牌表达
- 内容区承担信息密度和交互
- hero 可以个性化，但不要和正文区混成一个层次

实现建议：

- 纯视觉 hero：渐变 + 图片 + 蒙层
- 功能型 hero：标题 + 副标题 + 状态说明
- 高密度页面的 hero 高度要克制，避免首屏被装饰吞掉

### 2. 列表区

列表类页面建议统一遵循：

- 卡片之间留足纵向间距
- 同类卡片保持一致圆角和内边距
- 封面图、标题、地点、描述形成稳定信息顺序

推荐信息顺序：

1. 封面图
2. 标题
3. 地点/标签/状态
4. 简述
5. 二级操作

### 3. 输入区和表单区

以 `pages/ai-assistant/index.vue` 的输入区为参考：

- 输入容器用卡片式白底
- 发送按钮作为主操作，颜色明确
- 状态提示缩小处理，不和主输入抢权重
- 注意底部安全区和软键盘遮挡问题

后续攻略发布、登录注册、设置修改，也建议使用同样原则：

- 输入框视觉尽量统一
- 主按钮使用品牌主色
- 辅助说明采用 muted 风格

## 七、AI 助手页的组件规范补充

文件：`pages/ai-assistant/index.vue`

这是当前最完整的对话型页面，可作为以后聊天、问答、陪伴式页面的参考样板。

当前已形成的规范：

- 顶部用背景图 + 蒙层 + 居中头像标题
- AI 与用户消息分左右两侧布局
- 每条消息独立头像，不在底部输入栏重复头像
- 聊天气泡偏紧凑，适合高频阅读
- 快捷问题比正文轻一级，不抢会话主体
- 输入栏固定在底部，并做安全区避让

后续若新增类似页面，建议直接复用以下结构概念：

1. 头图区
2. 快捷入口区
3. 会话展示区
4. 固定输入区

## 八、尺寸与间距规范

项目当前以 `rpx` 为主。

建议延续以下层级：

- 页面左右边距：约 `24rpx` 到 `32rpx`
- 大卡片圆角：约 `24rpx` 到 `32rpx`
- 小胶囊圆角：`999rpx`
- 正文常规字号：约 `24rpx` 到 `30rpx`
- 分区标题字号：约 `34rpx` 到 `40rpx`
- 页面大标题可根据视觉需要单独放大

间距原则：

- 同组内容内部间距要小于组与组之间的间距
- 页面不要满屏都是大留白，也不要信息全部贴死
- 高密度页面优先压缩组件内部留白，而不是先删掉结构层级

## 九、后续新增组件的建议命名

为了保持项目可维护性，新增样式类名建议优先使用语义化命名，而不是纯视觉命名。

推荐：

- `hero-*`
- `section-*`
- `card-*`
- `meta-*`
- `tag-*`
- `status-*`
- `empty-*`
- `composer-*`

不推荐：

- `red-box`
- `big-text`
- `left-part`
- `style1`
- `new-card-2`

## 十、后续优化建议

当前项目已经有明确的暖色旅行视觉基础，但规范层还可以继续补强：

1. 把颜色变量进一步拆成语义变量，例如成功、警告、错误、信息色
2. 沉淀统一按钮规范，如主按钮、次按钮、弱按钮、文字按钮
3. 沉淀统一输入框规范，减少页面各自实现
4. 把常见空状态、加载状态、列表卡片进一步组件化
5. 为首页、AI 页、攻略页补一份更细的视觉差异规范，做到统一中保留个性

## 十一、推荐阅读顺序

如果后续开发者要快速理解项目样式与视觉体系，建议按以下顺序阅读：

1. `uni.scss`
2. `common/common.scss`
3. `components/AppTabBar.vue`
4. `components/CachedImage.vue`
5. `pages/home/index.vue`
6. `pages/ai-assistant/index.vue`
7. `md/样式实现说明.md`

这样可以先理解颜色和基础层，再理解组件，再理解页面实际落地方式。

# 遇见新疆_uniapp 样式实现说明

## 一、样式体系总览

这个项目的样式实现，主要基于以下几层组合完成：

1. `uni.scss`：定义全局主题变量
2. `common/common.scss`：定义全局基础样式和通用布局类
3. 各页面 `*.vue` 的 `style scoped lang="scss"`：负责页面级视觉实现
4. 少量全局组件样式：例如 `components/AppTabBar.vue`

项目没有单独引入大型 UI 框架，而是使用：

- `SCSS` 变量
- `uni-app` 原生组件样式
- 公共 class + 页面局部 class
- `rpx` 响应式尺寸

这套方式的特点是：

- 结构轻
- 页面可控性高
- 每个业务页面可以做较强的风格定制

## 二、主题变量如何实现

主题变量定义在：`uni.scss`

```scss
$theme-color: #c44536;
$theme-secondary: #e8a87c;
$theme-accent: #d4a574;
$theme-bg: #faf8f5;
$theme-text: #2d1810;
$theme-muted: #8b6f47;
$theme-border: rgba(196, 69, 54, 0.15);
```

这些变量构成了项目的基础视觉系统：

- `theme-color`：主色，偏砖红/新疆暖色调
- `theme-secondary`：辅助暖橙色
- `theme-accent`：强调金棕色
- `theme-bg`：页面背景米白色
- `theme-text`：主文字深棕色
- `theme-muted`：次级文字颜色
- `theme-border`：卡片边框/描边颜色

页面和组件通过 `@import '../../uni.scss'` 或 `@import '../uni.scss'` 来复用这些变量。

## 三、全局基础样式如何实现

全局样式集中在：`common/common.scss`

并在 `App.vue` 中全局引入：

```vue
<style lang="scss">
@import './common/common.scss';
</style>
```

这意味着所有页面默认共享同一套基础样式。

### 1. 页面基础风格

```scss
page {
  background: $theme-bg;
  color: $theme-text;
  font-size: 28rpx;
}
```

作用：

- 给整个应用统一背景色
- 统一默认文字颜色
- 给页面设置默认字号基准

### 2. 盒模型统一

```scss
view,
text,
input,
button {
  box-sizing: border-box;
}
```

作用：

- 避免宽高计算混乱
- 让 padding/border 更易控

### 3. 通用布局类

项目沉淀了几类高频 class：

- `.page-shell`：页面最外层容器
- `.page-scroll`：滚动区容器
- `.section`：统一左右边距
- `.card`：统一卡片视觉
- `.section-title`：统一标题字号与字重
- `.muted-text`：统一次级文字颜色
- `.pill`：统一胶囊按钮/标签基础形态
- `.hero-gradient`：统一顶部暖色渐变背景

例如：

```scss
.card {
  background: #ffffff;
  border: 2rpx solid $theme-border;
  border-radius: 28rpx;
  box-shadow: 0 16rpx 40rpx rgba(45, 24, 16, 0.08);
}
```

这就是项目里很多白色卡片视觉的来源。

## 四、尺寸和适配是怎么实现的

项目样式主要使用 `rpx`，这是 uni-app 常见的响应式单位。

优点：

- 适配不同屏宽较方便
- 安卓和小程序风格开发一致性较高

另外项目大量使用：

- `env(safe-area-inset-bottom)`
- `env(safe-area-inset-top)`

例如底部 tabbar 和输入栏，会通过安全区避免被刘海屏/底部手势条遮挡。

## 五、组件样式如何实现

### 1. 底部导航 `AppTabBar.vue`

文件：`components/AppTabBar.vue`

实现方式：

- 使用固定定位 `position: fixed`
- 外层 `.tabbar-wrap` 控制悬浮位置
- 内层 `.tabbar.card` 直接复用全局卡片样式
- 当前激活项通过 `.tab-item.active` 切换文字颜色

核心特点：

- 不是系统原生 tabBar，而是自绘 UI
- 视觉上更像悬浮胶囊导航

## 六、页面级样式是怎么实现的

每个页面通常使用：

```vue
<style scoped lang="scss">
@import '../../uni.scss';
</style>
```

这代表：

- 使用 `scss`
- 可以直接用全局颜色变量
- `scoped` 让样式主要作用于当前页面，降低污染

项目的主要视觉差异，基本都在页面局部样式里完成。

## 七、AI 助手页样式是怎么实现的

文件：`pages/ai-assistant/index.vue`

这个页面是当前项目里样式层次最完整的页面之一，包含：

- 顶部头图区
- 快捷问题区
- 聊天气泡区
- 底部输入栏

### 1. 顶部背景图 + 蒙层

通过 `background-image` 叠加实现：

```scss
.assistant-banner {
  background-image:
    linear-gradient(180deg, rgba(67, 36, 24, 0.28), rgba(67, 36, 24, 0.46)),
    url('/static/ai/linglu-background.jpg');
  background-size: cover;
  background-position: center;
}
```

作用：

- 底层是真实背景图
- 上层是半透明深色渐变蒙层
- 这样既能保留图片氛围，又不影响白字可读性

### 2. 头像圆形实现

AI 头像和用户头像都用：

```scss
.assistant-avatar,
.user-avatar {
  border-radius: 50%;
  overflow: hidden;
}
```

再配合 `image mode="aspectFill"`，实现：

- 始终圆形
- 图片裁切不变形

### 3. 头部居中布局

灵鹿头部使用纵向 flex：

```scss
.assistant-hero-row {
  display: flex;
  flex-direction: column;
  align-items: center;
  justify-content: center;
  text-align: center;
}
```

所以头像、标题、副标题都能保持居中。

### 4. 字体切换

灵鹿标题使用了偏中文书卷感字体：

```scss
.banner-title {
  font-family: 'STKaiti', 'KaiTi', 'FangSong', serif;
}
```

这里是字体栈回退策略：

- 系统有 `STKaiti` 就优先用
- 没有则回退 `KaiTi`
- 再回退 `FangSong`
- 最后回退 `serif`

### 5. 聊天气泡实现

聊天区域主要依赖 flex 布局：

- `.message-row`：单条消息横向容器
- `.message-row.mine`：用户消息靠右
- `.assistant-bubble`：AI 白底气泡
- `.user-bubble`：用户主色气泡

例如：

```scss
.message-row.mine {
  justify-content: flex-end;
}
```

表示用户消息整行右对齐。

而气泡本身通过：

- `padding`
- `border-radius`
- `box-shadow`
- 背景色差异

来形成聊天软件的视觉层次。

### 6. AI 流式输出效果

AI 回复不是一次性直接“蹦出”，而是通过 JS 分段渲染；样式上又配合：

- `.stream-cursor`
- `@keyframes cursorBlink`
- `.waiting-dot`
- `@keyframes waitingPulse`

实现“正在生成”“光标闪动”“思考中圆点跳动”的感觉。

也就是说，这一页的体验是：

- 逻辑层控制分段输出
- 样式层控制动画感和节奏感

### 7. 底部输入栏实现

底部输入区用固定定位：

```scss
.composer-wrap {
  position: fixed;
  left: 0;
  right: 0;
  bottom: 0;
}
```

然后再配合：

- 半透明背景
- 顶部分隔线
- 模糊背景 `backdrop-filter: blur(...)`

来模拟聊天软件底部发送栏。

## 八、攻略页样式是怎么实现的

文件：`pages/guides/index.vue`

攻略页的样式实现重点是：

- 顶部 hero 区
- 双列瀑布流
- 图文卡 / 文字卡 / 视频卡 三种卡片形态

### 1. 顶部区域

攻略页顶部使用全局 `.hero-gradient`，也就是统一暖色渐变背景。

这使得首页、攻略页、部分运营头部有统一品牌色调。

### 2. 双列瀑布流

页面不是直接用原生 masonry，而是：

- JS 先把数据分成左右两列
- 模板中两列分别渲染
- 卡片高度通过内容估算控制平衡

样式上再通过：

- 两列容器
- 卡片圆角
- 封面图遮罩
- 底部作者区

实现类似小红书的信息流效果。

### 3. 封面遮罩与标签

图文/视频卡片通常会叠加：

- cover 图
- `cover-mask`
- content/type/location 标签

这样可以保证文字在图片上仍可读。

### 4. 无图文字卡

没有图片时，不走封面区域，而是走 `text-note-shell` 这类纯文字卡。

这也是页面支持“纯文字笔记”的样式基础。

## 九、为什么这个项目看起来是统一风格

主要有四个原因：

1. 所有页面共用一套主题色变量
2. 全局有统一卡片、section、pill、标题样式
3. 主要页面都围绕暖色、米白、棕色体系展开
4. 页面虽然各自写 scoped 样式，但都遵循同一视觉基调

所以它不是“完全组件化设计系统”，而是：

- 一套轻量主题变量
- 一组公共基础类
- 每个页面再做局部增强

## 十、后续如果继续优化样式，建议怎么做

如果后面还要继续升级项目 UI，建议从下面几个方向继续整理：

### 1. 增加语义化设计变量

现在变量偏少，后续可以补：

- `success/warning/error`
- `surface/card/overlay`
- `shadow-sm/md/lg`
- `radius-sm/md/lg/xl`

### 2. 抽离更多通用 class

比如：

- 通用按钮类
- 通用输入框类
- 通用头像类
- 通用状态标签类

### 3. 抽离页面级样式规范

例如统一：

- 顶部 hero 高度区间
- 卡片间距区间
- 标题字号体系
- 页面空态样式

### 4. 建立真正的设计文档

可以进一步补成：

- 色板文档
- 字体层级文档
- 间距规范文档
- 组件视觉规范文档

## 十一、总结

这个项目的样式实现，本质上是：

- `uni.scss` 负责主题色
- `common/common.scss` 负责全局基础样式
- 页面 `scoped scss` 负责业务视觉表达
- 通过 `rpx + flex + card + gradient + shadow` 组合出统一风格

当前最典型的样式落地页面有：

- `pages/ai-assistant/index.vue`
- `pages/guides/index.vue`
- `components/AppTabBar.vue`

如果后续还要继续整理，我建议下一步专门再补一份：`颜色与组件规范说明.md`，这样整个项目的 UI 体系会更容易持续维护。

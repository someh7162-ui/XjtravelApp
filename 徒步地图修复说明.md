# 徒步地图修复说明

## 问题现象

- Android 徒步页地图黑屏，底部只显示纯黑背景。
- 原生 `plus.webview.create` 地图方案会压住页面上的 Vue 悬浮 UI，导致按钮和面板被遮挡。
- 兼容瓦片地图曾使用 `Promise.all + uni.downloadFile` 并发下载大量瓦片，在 Android 上容易触发网络并发过高，导致请求失败或加载异常。

## 根因分析

### 1. APP 端错误走了原生 Webview 地图路径

`pages/hiking/components/HikingMapStage.vue` 在 `#ifdef APP-PLUS` 下加载的是 `AppAmapWebview.vue`。

这个方案依赖 `plus.webview.create` 创建原生子 Webview。实际运行中：

- 子 Webview 路径加载不稳定，容易出现透明页或 404；
- 地图没正常渲染时，会直接露出底层黑色背景；
- 即使渲染成功，原生 Webview 也会盖在普通 Vue 视图上方。

### 2. 原生 Webview 天生会遮挡 Vue UI

`plus.webview.create` 创建的是原生层组件，层级高于普通页面节点，因此会遮住：

- 地图上的控制按钮
- 顶部状态面板
- 其他悬浮提示 UI

这与徒步页当前的交互设计冲突，不适合作为该页面的最终显示方案。

### 3. 兼容瓦片地图的预下载并发过高

`HikingTileMapCompat.vue` 原先会：

- 监听 `tiles`
- 对底图和注记图发起 `uni.downloadFile`
- 使用 `Promise.all` 一次性并发下载多张瓦片

Android 真机下，这种瞬时并发很容易把网络请求打爆，导致部分瓦片失败，反而比直接让 `image` 自己加载更不稳定。

## 本次修复

### 1. APP 端废弃 `AppAmapWebview.vue`

修改文件：`pages/hiking/components/HikingMapStage.vue`

- 保留 H5 端 `H5AmapTiandituMap.vue`
- APP 端改为使用 `HikingTileMapCompat.vue`
- 将 `mapCenter`、`mapScale`、`mapPolyline`、`mapMarkers`、`mapModeKey` 等参数继续透传

这样做的结果：

- 不再依赖原生子 Webview
- 不再出现原生层压住 Vue UI 的问题
- 地图和页面悬浮控件可以同时正常显示

### 2. APP 端地图支持判断改为兼容瓦片模式可直接显示

修改文件：`pages/hiking/components/HikingMapStage.vue`

- 新增 `hasMapSupport()`
- APP 端直接返回 `true`
- H5 端仍按原来的高德 H5 key 判断

这样可以避免 APP 端被高德 H5 key 逻辑误伤，确保瓦片兼容地图能直接渲染。

### 3. 删除瓦片预下载和本地缓存中转逻辑

修改文件：`pages/hiking/components/HikingTileMapCompat.vue`

已删除：

- `preloadTiles`
- `downloadTile`
- `resolveTileSrc`
- `localTileMap`
- `isDownloading`
- `watch(tiles, ...)` 预下载逻辑

现在改为：

- 底图 `<image>` 直接使用 `tile.baseUrl`
- 注记图 `<image>` 直接使用 `tile.annotationUrl`
- 交给 uni-app 原生 `image` 组件自行处理网络请求和缓存

这样做的结果：

- 避免了几十个 `downloadFile` 瞬间并发
- 减少 Android 网络层被掐断的风险
- 代码更简单，渲染链路更直观

## 当前结果

- Android 徒步地图已经可以显示
- 页面上的悬浮按钮、状态面板不再被地图遮挡
- 地图轨迹和定位点仍然可以继续叠加显示
- APP 端不再依赖高德 Webview 地图方案

## 后续建议

### 1. 如果还要继续增强稳定性

可以继续在 `HikingTileMapCompat.vue` 做：

- 瓦片范围裁剪
- 切图数量控制
- 缩放级别细化
- 失败瓦片重试

### 2. 如果未来要恢复真地图 SDK

建议优先考虑：

- 使用不会压住页面 UI 的官方组件方案
- 或单独设计一个原生全屏地图页，避免与当前 Vue 浮层混排

不要直接回到当前这种 `plus.webview.create` 嵌子地图的页面内混排模式。
